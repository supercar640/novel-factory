# Novel Forge Claude - 웹소설 작성 도구 구현 계획서

## 개요

PD(기획자)와 AI가 협업하여 웹소설을 기획하고 집필하는 인터랙티브 작성 도구.
PD의 의사결정(선정/보류/폐기/수정)을 중심으로 단계별로 진행되며, 총 4개의 Phase로 구성된다.

---

## Phase 1: 컨텍스트 수립 (방향성 선정 → 기획안 완성)

### 1-1. 방향성 제안

| 항목 | 내용 |
|------|------|
| **입력** | PD가 장르명, 연관 키워드, 분위기 등을 제시 |
| **처리** | AI가 해당 정보를 바탕으로 서로 다른 방향성 5개를 짧은 요약 형태로 생성 |
| **출력** | 번호가 매겨진 방향성 리스트 (각 2~3줄 요약) |

### 1-2. PD 의사결정 (방향성)

| 판정 | 설명 |
|------|------|
| **선정** | 5개 중 **1개만** 선택 가능. 선정 즉시 다음 단계로 이동 |
| **보류(Keep)** | 당장 선택하지 않지만 후보로 유지 |
| **폐기** | 후보에서 제거 |
| **리트라이** | 선정할 만한 항목이 없을 경우, 전체를 폐기하고 새로운 5개를 재생성 |

- 리트라이 시 PD에게 추가 키워드나 방향 힌트를 요청하여 품질 개선

### 1-3. 기획안 빌드업

| 항목 | 내용 |
|------|------|
| **입력** | 선정된 방향성 1개 |
| **처리** | AI가 구체적인 기획안으로 확장 (세계관, 주인공, 컨셉트, 플롯, 테마, 그 외 톤앤매너, 예상 분량 등) |
| **출력** | 구조화된 기획안 문서 |

### 1-4. PD 의사결정 (기획안)

| 판정 | 설명 |
|------|------|
| **선정(승인)** | 기획안 확정 → Phase 1 완료, 컨텍스트 확정 |
| **수정** | PD가 수정 지시 → AI가 반영 후 재제출 (반복 가능) |
| **폐기** | 기획안 폐기 → 1-1 단계로 복귀하여 방향성부터 재시작 |

### 1-5. 컨텍스트 확정
**선정(승인) 시**, AI는 확정된 기획안을 바탕으로 현재 작업 중인 디렉토리 하위에 기획된 소설의 제목(영어, 가칭)을 임의로 작성한 후 그 하위에 `context` 폴더를 생성하고 그 안에 아래 6개의 필수 마크다운 파일을 생성합니다:
        - character_profiles.md (주인공, 조연 등 캐릭터 프로필)
        - setting_world.md (시대/공간적 배경 및 고유 규칙)
        - concept.md (로그라인, 장르, 매력 포인트)
        - plot_outline.md (전체 시놉시스와 플롯 뼈대)
        - themes.md (스토리를 관통하는 테마와 메인 상징물)
        - tone.md (톤앤매너, 예상 분량 등 기타사항)

### Phase 1 완료 조건
- PD가 기획안을 **승인**하면 해당 기획안이 **컨텍스트**로 확정되며 Phase 2로 진행

## Phase 2: 전개 선정 (스토리 전개 옵션 제시 → 전개 확정)

### 2-1. 전개 옵션 생성

| 항목 | 내용 |
|------|------|
| **입력** | Phase 1에서 확정된 컨텍스트 |
| **처리** | AI가 **완전히 다른 방향성**을 가진 5개의 전개 옵션을 생성 |
| **출력** | 각 옵션에 아래 요소를 포함 |

**각 전개 옵션의 구성:**

```
옵션 N.
<text>방향성 요약 내용</text>
<probability>0.XX</probability>
```

- `<text>`: 해당 전개의 구체적 방향성 요약
- `<probability>`: 일반적인 소설 데이터베이스 기준 예상 등장 확률 (0.0 ~ 1.0)
  - 1.0에 가까울수록 뻔한 전개
  - 0.0에 가까울수록 독창적이고 기발한 전개

**샘플링 분포 규칙:**

| 구분 | 개수 | probability 범위 | 설명 |
|------|------|------------------|------|
| 일반 옵션 | 3개 | 0.90 이하 | 비교적 자연스러운 전개 범위에서 샘플링 |
| 희귀 옵션 | 2개 | 0.10 미만 | 꼬리 분포 영역, 매우 독창적이고 희귀한 전개 |

### 2-2. PD 의사결정 (전개)

| 판정 | 설명 |
|------|------|
| **선정** | 최소 1개 ~ **최대 3개** 선택 가능 |
| **보류(Keep)** | 당장 선택하지 않지만 후보로 유지 |
| **폐기** | 해당 옵션 제거 |
| **전체 폐기 후 리트라이** | 마음에 드는 옵션이 없으면 5개 전부 폐기 후 재생성 |
| **부분 재생성(reGen)** | 선정된 항목은 유지하고 미선정 항목만 폐기 후 새 옵션 재생성. 최대 3개 선정까지 반복 가능 |

### 2-3. 전개 선정 확인

- 전개 선정 후 AI가 반드시 확인: **"전개 선정을 종료하시겠습니까?"**
- PD가 종료를 승인하면 Phase 2 완료
- 추가 선정을 원하면 보류 목록에서 추가 선택 가능 (최대 3개 제한 유지)

### Phase 2 완료 조건
- PD가 1~3개의 전개를 선정하고, 종료를 승인하면 Phase 3으로 진행

## Phase 3: 집필 (소설 작성 → 승인)

### 3-1. 문체 설정

| 항목 | 내용 |
|------|------|
| **처리** | AI가 PD에게 참고할 문체(작가명, 작품명, 문체 설명 등)를 요청 |
| **입력** | PD가 문체 레퍼런스를 제공하거나 "없음"으로 응답 |
| **적용** | 문체가 있으면 해당 스타일을 모방하여 집필, 없으면 AI 기본 문체로 작성 |

### 3-2. 작성 모드 선택

PD가 **전개별로** 아래 두 가지 작성 모드 중 하나를 선택:

| 모드 | 설명 |
|------|------|
| **장면별 작성** | 하나의 장면(Scene)을 작성하고, 해당 장면이 끝나면 일단 종료하여 PD 피드백 수렴 |
| **1화 분량 작성** | 한 회차 분량(5,500자 이상)을 일괄 작성 |

- 전개가 1개면 모드 1개만 선택
- 전개가 복수(2~3개)면 각 전개마다 개별적으로 모드를 선택 가능
- 문체(style_reference)는 전체 공통으로 적용

### 3-3. 멀티 에이전트 모드 (전개가 복수인 경우)

- Phase 2에서 선정된 전개가 **2개 이상**일 경우 적용
- 각 전개에 대해 별도의 에이전트를 할당하여 **동시 병렬 작성**
- 각 에이전트는 동일한 컨텍스트와 문체 설정을 공유하되, **해당 전개에 설정된 작성 모드**와 서로 다른 전개 방향으로 집필
- PD는 각 에이전트의 결과물을 개별적으로 검토

### 3-4. PD 의사결정 (원고)

| 판정 | 설명 |
|------|------|
| **승인** | 원고 확정 → Phase 3 완료 |
| **수정** | PD가 수정 지시 → AI가 반영 후 재제출 (반복 가능) |
| **폐기** | 원고 폐기 → 3-3 단계로 복귀하여 재작성 |

### Phase 3 완료 조건
- PD가 원고를 **승인**하면 해당 회차 집필 완료 → Phase 4로 진행

---

## Phase 4: 퇴고 및 컨텍스트 갱신 (원고 확정 → 컨텍스트 업데이트 → Phase 2 복귀)

### 4-1. 퇴고 (교정·교열)

| 항목 | 내용 |
|------|------|
| **입력** | Phase 3에서 승인된 원고 + 현재 컨텍스트 |
| **처리** | AI가 원고를 재검토하여 문체 일관성, 오탈자, 문맥 흐름, 캐릭터 행동 일관성, 설정 충돌 등을 점검 |
| **출력** | 수정 사항이 표시된 퇴고 결과물 |

### 4-2. PD 의사결정 (퇴고)

| 판정 | 설명 |
|------|------|
| **승인** | 퇴고 완료 → 원고 최종 확정, 4-3 컨텍스트 갱신으로 이동 |
| **수정** | PD가 추가 수정 지시 → AI가 반영 후 재제출 (반복 가능) |
| **폐기** | 퇴고 결과 폐기 → Phase 3으로 복귀하여 재집필 |

### 4-3. 컨텍스트 갱신

| 항목 | 내용 |
|------|------|
| **입력** | 최종 확정된 원고 + 기존 컨텍스트 파일들 |
| **처리** | AI가 확정 원고에서 발생한 변경 사항을 분석하여 컨텍스트 파일 업데이트 |
| **대상** | `context/` 폴더 내 해당하는 파일들 (character_profiles.md, setting_world.md, plot_outline.md 등) |

**갱신 항목 예시:**
- 새로 등장한 캐릭터 또는 기존 캐릭터의 관계 변화 → `character_profiles.md`
- 새로 밝혀진 세계관 요소 또는 설정 변화 → `setting_world.md`
- 플롯 진행 상황 반영 → `plot_outline.md`
- 테마·복선 전개 추적 → `themes.md`

### 4-4. 컨텍스트 크기 관리

AI가 갱신 후 컨텍스트 전체 크기를 평가하여, AI 처리에 부담이 되는 수준이라고 판단될 경우 자동으로 압축 프로세스를 실행한다.

**압축 프로세스:**

| 단계 | 설명 |
|------|------|
| **1. 백업 생성** | 소설 디렉토리 하위에 `backup/context_v{N}/` 폴더를 생성하고, 현재 `context/` 파일들을 전부 복사 (N은 자동 증가 버전 번호) |
| **2. 요약본 재생성** | 기존 컨텍스트의 핵심 내용만 추출하여 압축된 요약본으로 `context/` 파일들을 교체 |
| **3. PD 확인** | AI가 PD에게 압축 사실과 요약본을 제시하고, PD가 승인하면 확정. 수정 요청 시 요약본 조정 |

**디렉토리 구조 예시:**
```
{소설제목}/
├── context/                    ← 현재 활성 컨텍스트 (압축 후 요약본)
│   ├── character_profiles.md
│   ├── setting_world.md
│   ├── concept.md
│   ├── plot_outline.md
│   ├── themes.md
│   └── tone.md
├── backup/
│   ├── context_v1/             ← 첫 번째 백업 (원본)
│   │   ├── character_profiles.md
│   │   └── ...
│   └── context_v2/             ← 두 번째 백업
│       └── ...
└── ...
```

### 4-5. Phase 2 복귀

- 컨텍스트 갱신(및 필요 시 압축)이 완료되면, **갱신된 컨텍스트를 기반으로 Phase 2로 복귀**하여 다음 회차 전개 선정을 시작
- 이 순환(Phase 2 → 3 → 4 → 2)이 연재가 끝날 때까지 반복

### Phase 4 완료 조건
- PD가 퇴고를 승인하고, 컨텍스트 갱신(필요 시 압축 포함)이 완료되면 Phase 2로 복귀

## 전체 워크플로우 요약

```
[Phase 1: 컨텍스트 수립] ─── 최초 1회만 실행
  PD 입력 (장르/키워드)
    → AI 방향성 5개 제안
    → PD 선정(1개) / 보류 / 폐기 / 리트라이
    → AI 기획안 빌드업
    → PD 승인 / 수정 / 폐기
    → 컨텍스트 확정

  ┌───────────────────────────────────────────────────┐
  │          ↓ (최초) 또는 ↑ (Phase 4에서 복귀)          │
  │                                                   │
  │  [Phase 2: 전개 선정]                                │
  │    컨텍스트 기반 전개 옵션 5개 생성                     │
  │      (일반 3개: prob ≤ 0.90 / 희귀 2개: prob < 0.10)  │
  │      → PD 선정(1~3개) / 보류 / 폐기 / 리트라이         │
  │      → 전개 확정                                     │
  │                    ↓                                │
  │  [Phase 3: 집필]                                     │
  │    문체 설정 → 작성 모드 선택 → 집필                    │
  │      → 멀티 에이전트 병렬 작성 (복수 전개 시)            │
  │      → PD 승인 / 수정 / 폐기                          │
  │      → 원고 확정                                     │
  │                    ↓                                │
  │  [Phase 4: 퇴고 및 컨텍스트 갱신]                      │
  │    AI 퇴고 (문체·설정 일관성 점검)                      │
  │      → PD 승인 / 수정 / 폐기                          │
  │      → 원고 최종 확정                                 │
  │      → 컨텍스트 파일 갱신                              │
  │      → (필요 시) 컨텍스트 백업 & 요약 압축              │
  │      → Phase 2로 복귀 ─────────────────┘             │
  └───────────────────────────────────────────────────┘
```

## 구현 시 핵심 원칙

1. **PD 중심 의사결정**: 모든 단계에서 AI는 제안하고, PD가 최종 판단
2. **반복 가능한 루프**: 각 단계에서 만족스러운 결과가 나올 때까지 리트라이 가능
3. **순환 구조**: Phase 2 → 3 → 4 → 2 사이클을 반복하며 연재를 이어감
4. **상태 관리**: 선정/보류/폐기 상태를 명확히 추적
5. **확률 기반 창의성 조절**: Phase 2에서 일반 전개와 희귀 전개를 분리하여 다양성 보장
6. **병렬 처리**: 복수 전개 시 멀티 에이전트로 동시 작성하여 효율 극대화
7. **컨텍스트 지속 관리**: 회차 진행에 따라 컨텍스트를 갱신하고, 크기 부담 시 백업 후 요약 압축
